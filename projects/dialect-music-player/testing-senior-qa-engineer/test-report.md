## 4. Bug 列表

### BUG-001: 方言搜索接口路由冲突（严重）

| 项目 | 内容 |
|------|------|
| **Bug ID** | BUG-001 |
| **严重程度** | 🔴 高（P0 阻塞） |
| **影响模块** | 方言模块 |
| **接口** | `GET /api/dialect/search` |
| **问题描述** | 方言搜索接口无法正常访问，调用 `/api/dialect/search?keyword=xxx` 时，FastAPI 将 `search` 误认为是 `{dialect_id}` 路径参数， |
| **复现步骤** | 1. 调用 `GET /api/dialect/search?keyword=成都`<br>2. 检查响应 |
| **实际结果** | 返回 `{"code":1201,"message":"方言不存在","data":null}` |
| **预期结果** | 返回搜索结果列表 |
| **根本原因** | FastAPI 路由定义顺序错误：`/search` 路由定义在 `/{dialect_id}` 之后，导致 `search` 字符串被当作 `dialect_id` 参数处理 |
| **修复建议** | 将 `@router.get("/search")` 移动到 `@router.get("/{dialect_id}")` **之前** 定义 |
| **状态** | 待修复 |

**代码位置**：`backend/routers/dialect.py`
```python
# ❌ 错误：动态路由在前
@router.get("/{dialect_id}")
async def get_dialect_detail(dialect_id: str):
    ...

# ❌ 搜索路由被拦截
@router.get("/search")
async def search_dialects(keyword: str):
    ...

# ✅ 正确：固定路由在前
@router.get("/search")
async def search_dialects(keyword: str):
    ...

@router.get("/{dialect_id}")
async def get_dialect_detail(dialect_id: str):
    ...
```

**BUG-001（方言搜索接口路由冲突）** 是一个**严重的设计缺陷**，会导致以下问题：
1. 用户无法使用搜索功能查找方言

3. 影响用户体验

### 6.3 修复建议

#### 立即修复（P0）
1. **调整路由顺序**：修改 `backend/routers/dialect.py`，将 `/search` 路由移到 `/{dialect_id}` 之前
2. **统一规范**：检查所有路由文件，确保固定路径路由定义在动态路径路由之前

#### 后续优化（P1）
1. 添加单元测试覆盖路由顺序问题
2. 补充音频文件测试数据
