## 1. 用户角色定义

### 1.1 普通员工（Employee）

**角色描述**：需要记录日常工作的普通员工

**职责**：
- 填报每日工时
- 查看和修改自己的工时记录
- 查看个人工时统计

**权限**：
| 权限 | 说明 |
|------|------|
| 填报工时 | ✅ 只能填报自己的 |
| 查看自己的记录 | ✅ |
| 编辑自己的记录 | ✅ |
| 删除自己的记录 | ✅ |
| 查看个人统计 | ✅ |
| 查看他人记录 | ❌ |
| 管理项目 | ❌ |
| 导出报表 | ❌ |

**典型画像**：
- 小王，28岁，软件开发工程师
- 每天需要在多个项目间切换
- 习惯在下班前填写当天工时
- 偶尔会忘记填报，需要补填

### 1.2 管理员（Admin）

**角色描述**：负责团队管理和数据导出的管理者

**职责**：
- 管理项目列表
- 查看团队工时数据
- 导出工时报表

**权限**：
| 权限 | 说明 |
|------|------|
| 填报工时 | ✅ |
| 查看所有记录 | ✅ |
| 管理项目 | ✅ 创建/编辑/归档 |
| 导出报表 | ✅ CSV/Excel |
| 查看团队统计 | ✅ |

**典型画像**：
- 李经理，35岁，项目经理
- 管理一个10人团队
- 每周五需要查看团队工时汇总
- 月底需要导出报表给财务

## 3. 数据实体

### 3.1 实体关系图（ER图）

```
┌──────────────┐       ┌──────────────┐       ┌──────────────┐
│    User      │       │  TimeEntry   │       │   Project    │
├──────────────┤       ├──────────────┤       ├──────────────┤
│ id (PK)      │──┐    │ id (PK)      │    ┌──│ id (PK)      │
│ username     │  │    │ userId (FK)  │◄───┘  │ name         │
│ password     │  └───►│ projectId(FK)│◄──────│ description  │
│ name         │       │ task         │       │ status       │
│ role         │       │ hours        │       │ createdAt    │
│ createdAt    │       │ date         │       └──────────────┘
└──────────────┘       │ note         │
                       │ createdAt    │
                       └──────────────┘

关系说明：
- User 1:N TimeEntry  （一个用户有多条工时记录）
- Project 1:N TimeEntry（一个项目有多条工时记录）
```

### 3.2 数据字典

#### 用户表 (users)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名（登录名） |
| password | VARCHAR(255) | NOT NULL | 密码（bcrypt加密） |
| name | VARCHAR(100) | NOT NULL | 员工姓名 |
| role | ENUM | NOT NULL, DEFAULT 'employee' | 角色：employee/admin |
| createdAt | DATETIME | NOT NULL | 创建时间 |

#### 项目表 (projects)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| name | VARCHAR(100) | NOT NULL | 项目名称 |
| description | TEXT | NULL | 项目描述 |
| status | ENUM | NOT NULL, DEFAULT 'active' | 状态：active/archived |
| createdAt | DATETIME | NOT NULL | 创建时间 |

#### 工时记录表 (time_entries)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| userId | INTEGER | FK, NOT NULL | 用户ID |
| projectId | INTEGER | FK, NOT NULL | 项目ID |
| task | VARCHAR(200) | NOT NULL | 任务名称 |
| hours | DECIMAL(4,1) | NOT NULL | 时长（小时） |
| date | DATE | NOT NULL | 工作日期 |
| note | TEXT | NULL | 备注说明 |
| createdAt | DATETIME | NOT NULL | 创建时间 |
| updatedAt | DATETIME | NULL | 更新时间 |

### 3.3 索引设计

| 表 | 索引 | 字段 | 用途 |
|------|------|------|------|
| time_entries | idx_user_date | userId, date | 按用户查询工时 |
| time_entries | idx_project_date | projectId, date | 按项目查询工时 |
| time_entries | idx_date | date | 按日期筛选 |
| projects | idx_status | status | 筛选活跃项目 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | < 2秒 | 首屏加载 |
| 接口响应时间 | < 1秒 | 95% 请求 |
| 并发用户数 | 50 | 同时在线 |
| 数据库查询 | < 500ms | 单次查询 |
| 文件导出 | < 10秒 | 1000条记录 |

### 4.2 安全需求

| 需求 | 措施 |
|------|------|
| 身份认证 | 用户名密码登录，Session/Token 验证 |
| 密码存储 | bcrypt 加密，salt rounds ≥ 10 |
| 权限控制 | 基于角色的访问控制（RBAC） |
| 数据隔离 | 用户只能访问自己的数据（管理员除外） |
| SQL注入防护 | 使用参数化查询 |
| XSS防护 | 输出转义，CSP策略 |

### 4.3 可用性需求

| 需求 | 要求 |

| 系统可用性 | 99%（工作日 9:00-18:00） |
| 数据备份 | 每日自动备份 |

### 4.4 兼容性需求

## 5. 业务规则

### 5.1 工时填报规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-001 | 单次填报时长必须 > 0 且 ≤ 24 小时 |
| BR-002 | 只能填报过去 7 天内的工时（防止追溯太久） |
| BR-003 | 同一天同一项目可以多次填报，系统自动累加展示 |
| BR-004 | 只能选择状态为"active"的项目填报 |
| BR-005 | 用户只能修改/删除自己的工时记录 |

### 5.2 权限规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-010 | 只有管理员可以创建/编辑/归档项目 |
| BR-011 | 只有管理员可以查看所有员工的工时 |
| BR-012 | 只有管理员可以导出工时数据 |

### 5.3 数据规则

| 规则编号 | 规则描述 |
|----------|----------|
| BR-020 | 归档项目后，历史工时记录保留，但不可再填报 |
| BR-021 | 删除工时记录为物理删除（不做软删除） |
| BR-022 | 用户账号创建后不可删除，只能停用（预留） |

## 6. 输入输出规范

### 6.1 输入校验规则

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 项目 | 必选，必须是active状态 | "请选择有效的项目" |
| 任务 | 必填，1-200字符 | "请输入任务名称" |
| 时长 | 必填，0 < hours ≤ 24，支持1位小数 | "请输入有效时长（0-24小时）" |
| 日期 | 必填，不能超过今天，不能早于7天前 | "请选择有效的日期" |
| 备注 | 可选，最多500字符 | "备注不能超过500字符" |

### 6.2 输出格式

#### 日期时间格式
- 日期：`YYYY-MM-DD`（如 2026-03-19）
- 时间：`YYYY-MM-DD HH:mm:ss`
- 显示：`03月19日 10:30`

#### 数字格式
- 时长：保留1位小数（如 4.5h）
- 统计汇总：保留1位小数

## 7. 异常处理

### 7.1 业务异常

| 异常场景 | 处理方式 |
|----------|----------|
| 填报日期超出范围 | 前端限制选择，后端返回错误 |
| 项目已归档 | 不在选择列表显示，后端返回错误 |
| 时长输入非法 | 前端校验 + 后端校验 |
| 修改他人记录 | 后端权限校验，返回403 |

### 7.2 系统异常

| 异常场景 | 处理方式 |
|----------|----------|
| 数据库连接失败 | 返回500，提示"系统繁忙，请稍后重试" |
| 导出文件过大 | 限制导出范围，分批导出 |
| 并发写入冲突 | 乐观锁重试 |
