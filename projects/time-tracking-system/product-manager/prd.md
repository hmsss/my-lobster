## 1. 背景与问题陈述

### 为什么做这个产品？
企业需要追踪员工在各个项目上投入的时间，用于：
- 项目成本核算
- 员工绩效评估
- 资源调配决策
- 客户计费依据

### 用户痛点
1. **员工**：手动填写 Excel 表格繁琐，容易遗漏，提交不及时
2. **管理员**：汇总统计困难，数据分散，难以快速生成报表
3. **财务**：缺少标准化的工时数据，影响成本核算效率

### 目标用户
- **普通员工**：需要记录每日工作时间的员工
- **项目经理**：需要查看项目工时投入的管理者
- **HR/财务**：需要导出工时数据进行核算的人员

## 2. 目标与成功指标

| 目标 | 指标 | 目标值 | 度量窗口 |
|------|------|--------|----------|
| 提高填报效率 | 平均填报时间 | < 2分钟/天 | 上线后 30 天 |
| 提高填报率 | 日填报完成率 | > 90% | 上线后 14 天 |
| 减少统计工作量 | 报表生成时间 | < 10秒 | 上线后 7 天 |
| 数据准确性 | 填报错误率 | < 5% | 上线后 30 天 |

## 3. Non-Goals（不做的事）

| 不做的事 | 原因 | 重新考虑条件 |
|----------|------|--------------|
| 移动端 App | V1 聚焦 Web 版本，移动端用响应式 | 用户数据显示移动端使用 >30% |
| 考勤打卡集成 | 需要对接考勤系统，复杂度高 | 客户明确需求 |
| 审批流程 | 简化 V1 范围 | 客户需要管控工时准确性 |
| 多租户/SaaS | 单机部署，专注单企业场景 | 有商业化需求 |
| 项目管理功能 | 专注工时填报，不替代 PM 工具 | 客户需要一体化方案 |
| 自动工时追踪 | 技术复杂，涉及隐私 | 用户反馈手动填报负担重 |

## 4. 用户故事

### 故事 1：填报工时
**作为** 一名员工
**我希望** 快速填写我今天在各个项目上的工作时间
**以便** 记录我的工作投入

**验收标准**：
- [ ] 选择项目（下拉列表）
- [ ] 选择/输入任务名称
- [ ] 输入工作时长（小时，支持小数如 0.5）
- [ ] 选择日期（默认今天）
- [ ] 填写备注说明（可选）
- [ ] 提交后显示成功提示
- [ ] 同一天同一项目可多次填报，自动累加

### 故事 2：查看我的工时记录
**作为** 一名员工
**我希望** 查看我历史填报的工时记录
**以便** 回顾和核对

**验收标准**：
- [ ] 列表展示所有工时记录
- [ ] 显示：日期、项目、任务、时长、备注
- [ ] 支持按日期范围筛选
- [ ] 支持按项目筛选
- [ ] 分页显示（每页 20 条）

### 故事 3：编辑/删除工时记录
**作为** 一名员工
**我希望** 修改或删除我填错的工时记录
**以便** 保持数据准确

**验收标准**：
- [ ] 每条记录有编辑和删除按钮
- [ ] 编辑时预填充原有数据
- [ ] 删除前有确认提示
- [ ] 只能修改自己的记录

### 故事 4：查看个人工时统计
**作为** 一名员工
**我希望** 查看我的工时统计
**以便** 了解我的时间分配

**验收标准**：
- [ ] 按周/月切换统计周期
- [ ] 显示总工时
- [ ] 按项目分组的工时占比（饼图或柱状图）
- [ ] 每日工时趋势图

### 故事 5：管理员查看团队工时
**作为** 一名管理员
**我希望** 查看团队成员的工时数据
**以便** 了解项目投入情况

**验收标准**：
- [ ] 查看所有员工的工时记录
- [ ] 按员工筛选
- [ ] 按项目筛选
- [ ] 按日期范围筛选
- [ ] 显示团队总工时

### 故事 6：导出工时报表
**作为** 一名管理员
**我希望** 导出工时数据
**以便** 用于财务核算或汇报

**验收标准**：
- [ ] 支持导出 CSV 格式
- [ ] 支持导出 Excel 格式
- [ ] 可选择导出范围（日期、员工、项目）
- [ ] 导出文件包含完整字段

### 故事 7：项目管理
**作为** 一名管理员
**我希望** 管理项目列表
**以便** 员工填报时选择

**验收标准**：
- [ ] 创建新项目（名称、描述）
- [ ] 编辑项目信息
- [ ] 归档/激活项目
- [ ] 归档后的项目不在填报列表显示

### 6.2 关键交互流程

#### 工时填报流程
```
用户进入填报页
    ↓
选择项目（必填）
    ↓
输入任务名称（必填）
    ↓
输入时长（必填，校验 >0 且 ≤24）
    ↓
选择日期（默认今天，可选过去7天）
    ↓
填写备注（可选）
    ↓
点击提交
    ↓
显示成功提示 + 刷新今日统计
```

#### 工时列表交互
```
进入"我的记录"页面
    ↓
默认显示本月记录
    ↓
可筛选：日期范围、项目
    ↓
点击编辑 → 弹出编辑弹窗
点击删除 → 确认后删除
```

## 7. 接口需求

### 7.1 工时模块

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/timesheet` | POST | 提交工时 |
| `/api/timesheet` | GET | 获取工时列表（支持筛选） |
| `/api/timesheet/:id` | PUT | 更新工时 |
| `/api/timesheet/:id` | DELETE | 删除工时 |
| `/api/timesheet/stats` | GET | 获取个人工时统计 |

**请求/响应示例**：

```json
// POST /api/timesheet
{
  "projectId": 1,
  "task": "功能开发",
  "hours": 4.5,
  "date": "2026-03-19",
  "note": "完成用户模块"
}

// Response
{
  "code": 0,
  "message": "提交成功",
  "data": {
    "id": 123,
    "projectId": 1,
    "projectName": "项目A",
    "task": "功能开发",
    "hours": 4.5,
    "date": "2026-03-19",
    "note": "完成用户模块",
    "createdAt": "2026-03-19T10:30:00Z"
  }
}
```

### 7.2 项目模块

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/projects` | GET | 获取项目列表 |
| `/api/projects` | POST | 创建项目（管理员） |
| `/api/projects/:id` | PUT | 更新项目（管理员） |
| `/api/projects/:id/archive` | POST | 归档项目（管理员） |

### 7.3 统计模块

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/stats/personal` | GET | 个人工时统计 |
| `/api/stats/team` | GET | 团队工时统计（管理员） |
| `/api/export` | GET | 导出工时数据（管理员） |

## 8. 数据需求

### 8.1 核心数据实体

```
用户 (User)
├── id: 主键
├── username: 用户名
├── password: 密码（加密）
├── name: 姓名
├── role: 角色（employee/admin）
└── createdAt: 创建时间

项目 (Project)
├── id: 主键
├── name: 项目名称
├── description: 描述
├── status: 状态（active/archived）
└── createdAt: 创建时间

工时记录 (TimeEntry)
├── id: 主键
├── userId: 用户ID
├── projectId: 项目ID
├── task: 任务名称
├── hours: 时长（小时）
├── date: 日期
├── note: 备注
└── createdAt: 创建时间
```

### 8.2 数据关系
- User 1:N TimeEntry（一个用户多条工时）
- Project 1:N TimeEntry（一个项目多条工时）
