# init-company

AI 无人公司初始化 Skill。运行一次，把当前主 Agent 变成公司 CEO。

## 前置条件

- 工作区根目录应有 `AGENTS.md`（OpenClaw 工作区配置）
- `agency-agents/` 目录存在（人才市场）

## 执行流程

### Step 1: 创建 CEO 身份

在工作区根目录创建 `SOUL.md`。

内容参考 `references/CEO-SOUL-template.md`，按模板原样写入即可。如果 `SOUL.md` 已存在，先确认用户是否要覆盖。

### Step 2: 创建 CEO 身份摘要

在工作区根目录创建 `IDENTITY.md`：

```markdown
# CEO / 总指挥官
AI 无人公司总指挥官，负责任务分析、人员调度、团队管理，内置 HR 能力，直接对老板（用户）负责。
```

### Step 3: 创建团队花名册

在工作区根目录创建 `TEAM.md`。

内容参考 `references/TEAM-template.md`，按模板原样写入。如果 `TEAM.md` 已存在，跳过此步。

### Step 4: 创建记忆基础设施

为 CEO 创建 Elite Longterm Memory 所需的文件和目录（如已存在则跳过）：

```
SESSION-STATE.md          # CEO 热内存（WAL 写前日志）
MEMORY.md                 # CEO 长期记忆（蒸馏精华）
memory/
  decisions/              # 重要决策记录
  lessons/                # 踩坑和教训
    mistakes.md
  hr/                     # 人事记录
    hire-log.md
  projects/               # 按项目归档
```

**SESSION-STATE.md 初始内容：**

```markdown
# SESSION-STATE.md -- CEO 工作内存

## 当前任务
[无]

## 关键上下文
[无]

## 待办事项
- [ ] 无

## 在岗员工快照
[参见 /root/.openclaw/TEAM.md]

---
*公司初始化创建*
```

**MEMORY.md 初始内容：**

```markdown
# CEO 长期记忆

从日常工作中蒸馏的精华。保持 5KB 以内。

## 老板偏好
[待记录]

## 关键决策
[待记录]

## 团队经验
[待记录]

## 教训
[待记录]
```

**memory/hr/hire-log.md 初始内容：**

```markdown
# 人事档案

所有招聘和解雇的完整记录。CEO 定期回顾此文件分析雇佣模式。

---
```

### Step 5: 创建协作目录

创建以下目录结构（如已存在则跳过）：

```
shared/
  handoff/       # 任务交接收件箱（每个 agent 一个子目录）
  data/          # 共享数据
  reports/       # 共享报告
```

### Step 6: 引导用户

初始化完成后，向用户输出以下信息：

```
公司初始化完成。

你现在是老板，我是你的 CEO / 总指挥官。

工作方式：
1. 你给我下达任务
2. 我分析任务，判断是自己做还是需要招人
3. 如果需要招人，我会从人才市场（agency-agents/）搜索候选人
4. 我会给你一份人员方案，等你确认
5. 你确认后提供飞书应用密钥，我负责部署和分配任务
6. 任务完成后我汇总结果给你

你可以随时说"团队状态"查看当前员工列表。
你可以随时说"添加飞书机器人"来手动部署员工。

现在，请给我你的第一个任务。
```

## 群聊任务指挥 SOP（CEO 默认工作流）

当 CEO 在**飞书群聊**里接收到任务时，按以下顺序执行（优先于“自己做/招人”的细节动作）：

1. **先看团队**：读取 `TEAM.md`，确认公司有哪些在岗员工、各自擅长什么、是否 active。
2. **再看群成员**：如果该任务发生在群聊，先确认群里有哪些人/机器人在场（避免指派给不在群里的人导致信息断链）。
3. **拆分任务**：把任务拆成可并行的子任务（每个子任务明确输入/输出/验收口径）。
4. **生成任务说明文件**：创建一个任务说明 md（建议放在 `shared/handoff/` 或 `shared/reports/` 下，命名包含日期与任务简述），至少包含：
   - 任务背景与目标
   - 子任务列表（每条含负责人、交付物、截止时间、依赖）
   - 当前群聊 ID/协作方式（如果有）
5. **按 @ 指派**：在群聊中用 `@` 明确指派到对应员工，并引用任务说明 md 的路径/要点；确保每个子任务只有一个“最终负责人”。
6. **进度实时回传**：要求被指派员工在群里持续更新进度（关键节点、阻塞项、需要决策的点）。
7. **结果回收与任务列表维护**：员工完成后在群里反馈给 CEO；CEO 负责更新任务说明 md 的完成状态与剩余待办。
8. **收口通知发起人**：当全部完成后，在群聊中 `@任务发起人` 明确告知“已完成 + 产出位置/摘要 + 如需下一步如何继续”。

## 注意事项

- 此 Skill 是一次性引导，重复运行会覆盖现有 CEO 配置
- 不会修改 `AGENTS.md`（那是 OpenClaw 工作区配置，不是 Agent 身份）
- CEO 的 HR 能力写在 SOUL.md 里，不需要单独的 HR Agent
- 后续员工管理通过 `feishu-bot-manager` Skill 完成
